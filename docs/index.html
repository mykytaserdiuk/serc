Introduction

SER is a small interpreted programming language written in Go.

The goal of SER is to provide a simple syntax and lightweight runtime while still supporting functions, structures, modules, and native Go extensions.

Hello World

func main():
    print("Hello SER!\n")
end
Every program starts executing from the main() function.

Syntax

Comments

Comments start with ;. Everything after ; until the end of line is ignored. 

; this is a comment

func main():
    print("Hello")
end

Variables

Variables are created using def. 

func main():
    def name = "Oleg"
    def age = 20

    print(name)
end

Assignment

Existing variables can be changed without def. 

func main():
    def x = 0
    x = x + 1
end

Data Types

Type Description
Int Integer numbers
String Text values
Bool Boolean values
Null No value
Object Structures and complex values

Functions

Functions are declared using func. 

func hello():
    print("Hello")
end

Function arguments

func send(user):
    printf("%+v\n", user)
end

Return values

func getName():
    return "SER"
end

Functions without a return value return Null.

Conditions

Conditions use if and then. 

func main():
    if (5 + 2 <> 6) then
        print("true")
    end
end

Comparison operators

Operator Description
< Less than
> Greater than
<= Less or equal
>= Greater or equal
== Equal
<> Not equal

Loops

SER currently supports while loops. 

func main():
    def x = 0
    while(x < 10):
        printf("%+v\n", x)
        x = x + 1
    end
end

Structures

Structures allow creating custom objects.

Creating structure

struct User (
    first_name
    lastName
)

Creating object

func main():
    def user = User(
        first_name: "Oleg"
        lastName: "Vinnic"
    )
end

Accessing fields


user.first_name = "Daniel"
print(user.first_name)

Modules & Native Extensions

Modules allow splitting programs into separate files and extending the language using Go.

1. SER Modules

Standard modules are stored inside the modules/ directory as .ser files.

Example structure: 
project/
    main.ser

    modules/
        math.ser

Importing a standard module:

use math

Usage example:

; modules/math.ser
func add(a, b):
    return a + b
end

; main.ser
use math
func main():
    printf("%+v\n", math.add(1,5))
end

2. Native Go Built-ins

SER now supports Native Go Extensions. You can write high-performance built-in functions directly in Go and expose them to the SER runtime.

All native built-in methods and definitions are located inside the /builtin/ directory of the project. This is also the best place to look for practical examples and see how to create and register your own custom native modules.

This extension system is ideal for adding complex algorithms, file I/O, or networking features to the language using Go's ecosystem.

Example of defining a native function in Go:

// Inside the Go interpreter codebase
func HttpServe(args []ast.Value) ast.FuncResult {
	port := args[0]
	if port.Type != ast.StringValue {
		panic("RUNTIME ERROR: unsoportet 'http.serve' type")
	}
	err := http.ListenAndServe(port.Data.(string), nil)
	if err != nil {
		return ast.FuncResult{
			Value: ast.StringLiteral{
				Value: err.Error(),
			},
		}
	}
	return ast.FuncResult{
		Value: ast.NumberLiteral{Value: 0},
	}
}

Module Aliases

Any module (SER or Native) can have an alias. The alias becomes the namespace identifier inside the program. 

use math "calculator"

func main():
    calculator.add(1,5)
end
